# Zeratul

星际酒馆 PVP 模式的泽拉图预言牌推理工具。项目使用原生 `HTML + CSS + JavaScript` 实现，无框架、无构建链路、无后端依赖。

## 项目简介

本项目用于辅助玩家记录“进场卡牌”和“是否获得隐刀”的反馈，并基于泽拉图技能规则对核心包中的预言牌进行排除、收敛和推荐下一步试探卡牌。

当前实现是一个纯静态前端页面（V3.0.1）：

- 页面入口：`index.html`
- 业务逻辑：`app.js`
- 样式文件：`styles.css`
- 卡牌数据：`resource/data/*.js`

## 已实现功能

### 1. 卡包加载与数据管理

- 启动时默认加载核心包 `core`
- 额外卡包通过左侧复选框启用或停用
- 卡牌数据通过动态插入 `<script>` 标签异步加载到 `window._packData`
- 已加载脚本会缓存，避免重复注入
- 切换卡包后清空当前记录并重新计算候选牌
- 当前可选扩展包：`pack2`、`pack4`、`pack5`、`pack6`、`pack7`、`pack9`、`pack10`、`packDuo1`
- `pack1`、`pack3` 数据文件保留，但未在当前卡包选择中启用

### 2. 进场卡牌筛选与录入

- 按种族筛选可选卡牌
- 按等级筛选可选卡牌
- 展示符合条件的卡牌按钮
- 通过“奇数 / 偶数或0”记录本次是否获得隐刀
- 已进场卡牌不可重复录入
- 进场按钮带点击动画反馈

### 3. 历史记录管理

- 表格展示每次进场记录
- 展示卡牌名、种族、等级、单位数、价值、隐刀反馈
- 支持删除单条记录
- 支持清空全部记录

### 4. 候选预言牌计算

- 仅从核心包卡牌中筛选预言牌候选项
- 依据历史记录校验候选牌是否一致
- “接近”判定规则：
  - 种族相同
  - 初始单位数相同
  - 价值差值小于等于 `200`
- 支持按种族、单位数、价值排序
- 显示候选牌数量统计
- 支持手动排除候选牌
- 支持恢复全部排除

### 5. 约束推断

- 从“奇数”记录中提取正向约束
- 从“偶数 / 0”记录中提取排除约束
- 对种族、单位数、价值三个维度做约束传播
- 维度状态包括已锁定、已缩小、未知
- 约束面板按状态展示当前推断结果

### 6. 精准预测与下一步推荐

- 根据候选集计算收敛进度
- 估算收敛到唯一候选所需步数
- 对未使用卡牌计算信息增益
- 推荐最适合继续试探的卡牌
- 支持按等级过滤推荐池
- 候选只剩一张时展示锁定结果
- 无候选牌时展示警告状态

### 7. 弹窗与顶部入口

- 预言确认弹窗支持确认、取消、关闭、遮罩关闭
- 打开页面时展示更新日志弹窗
- “今日关闭”会写入 `localStorage.zeratul_update_dismissed`
- 顶部铃铛按钮可手动打开更新日志
- 顶部礼物按钮可手动打开福利弹窗
- 福利弹窗支持复制兑换口令
- 福利弹窗不会在页面打开时自动展示

### 8. 界面与响应式

- 三栏布局：卡包选择、操作记录、预测候选
- 星际争霸2 神族风格视觉
- 使用 `border-image` 实现纹理边框
- 支持自定义滚动条
- 支持帮助提示气泡
- 适配平板和手机宽度

## 模块说明

### `index.html`

负责页面结构：

- 顶部标题栏
- 福利按钮和更新日志按钮
- 工具说明与作者外链
- 左侧卡包选择区
- 中部进场控制区和历史记录表格
- 右侧精准预测面板和候选牌表格
- 预言确认弹窗
- 更新日志弹窗
- 福利弹窗

### `app.js`

负责全部业务逻辑，采用 IIFE 组织：

- `PACK_REGISTRY`：当前启用的卡包注册表
- `loadPackScript()`：动态加载卡包脚本
- `App.state`：全局应用状态
- `cacheEls()`：DOM 缓存
- `bindEvents()`：基础事件绑定
- `initUpdateModal()`：更新日志弹窗
- `initBenefitModal()`：福利弹窗
- `addGuess()` / `removeGuess()`：记录管理
- `excludeCandidate()` / `restoreAllExcluded()`：候选排除
- `candidateConsistent()` / `getCandidates()`：候选计算
- `buildConstraints()`：约束推断
- `calcRecommendations()`：推荐牌计算
- `renderAll()`：整体渲染

当前状态持久化接口 `persist()` 与 `restoreState()` 已预留，尚未实现。

### `styles.css`

负责视觉样式与响应式布局：

- 主题变量
- 三栏布局
- 筛选按钮
- 反馈按钮
- 候选牌标签
- 约束摘要
- 精准预测面板
- 表格布局
- 模态框
- 提示气泡
- 滚动条
- 按钮动画
- 响应式断点

### `resource/data/*.js`

负责卡牌数据注册。每个文件都会向全局对象写入如下结构：

```js
window._packData["packKey"] = {
  name: "卡包名",
  cards: [
    { id, race, level, number, value }
  ]
};
```

## 构建与运行方式

本项目没有打包、编译、测试或发布脚本，属于直接运行的静态页面项目。

### 本地运行

可使用任一静态文件服务器打开项目根目录：

```powershell
cd D:\Works\Zeratul
python -m http.server 8080
```

然后访问 `http://localhost:8080`。

也可以直接打开 `index.html`。由于项目会通过脚本标签动态加载数据文件，更推荐通过本地静态服务器访问。

### 构建方式

- 无 `npm` / `pnpm` / `yarn`
- 无 `package.json`
- 无 bundler
- 无转译流程
- 无 CI 配置

当前“构建”方式就是直接托管静态文件。
